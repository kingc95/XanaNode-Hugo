# XanaNode Proposals

XanaNode Proposals, or XNPs, are design records for protocol changes.

Use proposals for changes that affect:

  • canonical schemas
  • federation behavior
  • namespace rules
  • compatibility levels
  • transclusion
  • tumbler addressing
  • Git-backed substrates
  • derived inverse relationships
  • standards alignment profiles
  • federation discovery and trust
  • Studio local-first authoring
  • governance

Proposal statuses:

  • Draft
  • Review
  • Accepted
  • Rejected
  • Superseded
  • Experimental
