# XanaNode Governance

XanaNode governance defines how the protocol evolves while allowing independent substrates to remain sovereign.

The governance model is intentionally minimal:

1. Maintain a small stable core. 2. Allow namespaced extensions. 3. Preserve disagreement rather than deleting it. 4. Treat mappings, disputes, and merge results as knowledge. 5. Avoid centralizing truth in one moderation team.

Governance applies to the protocol repository. It does not dictate how every independent substrate must moderate its own content.
